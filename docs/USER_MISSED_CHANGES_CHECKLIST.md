# Чек-лист по «снесённым» изменениям (контрольный список пользователя)

Статусный формат:
- `[x]` реализовано в коде;
- `[~]` реализовано частично / нужно стендово подтвердить;
- `[ ]` не закрыто.

## 1) Загрузка и оффлайн-пересчет
- [x] При загрузке нового `CSV` в температурной компенсации данные и итог оффлайн-анализа пересчитываются.
- [x] При загрузке `all_nodes.csv` верхний селектор узлов (CAN-узел) не засоряется узлами из файла; отдельный селектор наборов находится внутри temp-comp блока.

## 2) Чтение из МК и лаги
- [x] Сильные лаги при «Прочитать из МК» снижены за счет очереди DID, ограниченного режима чтения и оптимизаций перерисовки.
- [x] Чтение параметров только для активного режима компенсации (mode-aware очередь DID), а не сразу всех.

## 3) UI калибровки
- [x] Калибровка 0%/100% вынесена в отдельный спойлер.
- [x] Цвета кнопок приведены к спокойной схеме по смыслу действий.
- [x] Блок «Итог офлайн-анализа» минимизирован (компактнее, меньше «полотна» текста).
- [x] «Быстрое превью линейного режима» вынесено в отдельный спойлер и по умолчанию закрыто.
- [x] Высота полей K0/K1 в превью увеличена/выровнена (текст умещается корректнее).
- [x] Блоки temp-comp визуально разделены (карточки/фон/границы/спойлеры), чтобы не сливались.

## 4) Алгоритм и прикладное поведение
- [~] Случай «после линейного режима на МК на комнатной температуре ~18%» разобран: требуется стендовая проверка по текущим DID узла и сценарию применения K1/K0.
- [x] Добавлено отдельное чтение K0/K1 из расширенных настроек.
- [x] Рекомендации переведены в строгий оффлайн-режим «только по CSV выбранного узла» (без влияния текущих DID на рекомендацию).
- [x] Добавлена кнопка «Подогнать K0 к 0% при текущей температуре» (автоматический расчет по текущим DID).

## 5) Форматы файлов и кодировки
- [x] Запись CSV выполняется только в `UTF-8`.
- [x] Добавлена загрузка не только `CSV`, но и `XLSX` для анализа и расчета `K0/K1`.

## 6) Пояснение по K0 (обязательная памятка)
- [~] Пояснение «как считается K0 сейчас и как применять на практике» дано, но нужно подтвердить на вашем стенде финальную процедуру для реального узла:
  - оффлайн `K1` по CSV выбранного узла;
  - запись `K1` в МК;
  - точечная автоподстройка `K0` кнопкой «Подогнать K0 к 0% при текущей температуре».

## Порядок дальнейшей работы по этому листу
1. Закрыть стендовую проверку пунктов с меткой `[~]`.
2. Зафиксировать фактические результаты «ожидание/факт» для `0x6A`, `0x8A`, `all_nodes`.
3. После подтверждения перевести оставшиеся `[~]` в `[x]`.
