# K0/K1: практический алгоритм настройки и причина отклонения на комнате

## Коротко
- `K1` — наклон температурной компенсации. Его рекомендуем считать строго оффлайн по CSV/XLSX конкретного узла.
- `K0` — вертикальное смещение (сдвиг уровня вверх/вниз). Его удобно уточнять на стенде в текущей точке температуры.

## Почему после записи линейных коэффициентов может быть +18% на комнатной температуре
- Оффлайн-рекомендация по CSV корректно выравнивает тренд по температуре, но не обязана идеально попадать в «нулевую» точку именно текущего стенда.
- В реальной проверке на узле может отличаться базовая точка (смещение), поэтому нужен отдельный шаг подстройки `K0` по текущим DID.
- Если 0%/100% в МК устарели или не соответствуют текущей физике, ошибка по уровню дополнительно растет.

## Рекомендуемая последовательность (понятная оператору)
1. Загрузить CSV/XLSX и выбрать нужный набор узла в блоке температурной компенсации.
2. Записать рекомендованный `K1` (и, при необходимости, связанные параметры режима).
3. Нажать кнопку `Подогнать K0 к 0% (текущая T)` для автоподстройки смещения в текущей температурной точке.
4. Снова прочитать параметры текущего режима и проверить колонку «Топливо».
5. Если отклонение осталось большим, перепроверить корректность 0%/100% и повторить шаги 2-4.

## Важно
- Оффлайн-рекомендация и онлайн-подгонка не конфликтуют: это два последовательных шага одной процедуры.
- Для режима `0 (single)` критично сначала зафиксировать `K1`, и только потом подгонять `K0`.
